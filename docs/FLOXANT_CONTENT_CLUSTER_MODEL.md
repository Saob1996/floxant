# FLOXANT Content-Cluster-Modell

Stand: 2026-05-05

Ziel dieser Datei ist ein stabiler Content-Plan ohne neue Doorway Pages. Neue URLs entstehen nur, wenn Suchintention, Inhaltstiefe, echte Leistung und interne Verlinkung stark genug sind. Standardfall: bestehende Geldseiten mit Modulen, FAQ und internen Links staerken.

## Bewertungslogik

| Kriterium | Bedeutung |
| --- | --- |
| Hauptseite | Primaere Zielseite fuer Ranking und Conversion |
| Support-Themen | Themen, die die Hauptseite staerken |
| Umsetzung | Modul, FAQ, Guide oder keine Umsetzung |
| CTA | Naechster sinnvoller Kundenschritt |
| Kannibalisierungsrisiko | Risiko, dass mehrere Seiten fuer dasselbe Keyword konkurrieren |

## Cluster 1: Umzug Regensburg

| Feld | Entscheidung |
| --- | --- |
| Hauptseite | `/umzug-regensburg` |
| Suchintention | Kunde sucht ein Umzugsunternehmen in Regensburg und will Ablauf, Preislogik, Zusatzservices und Anfrageweg verstehen. |
| Support-Themen | Umzug mit Reinigung, Schluesseluebergabe, Halteverbotszone, Uebergabevorbereitung, Entruempelung, Umzug innerhalb Bayern, Foto-Pruefung, Umzugspreis-Faktoren. |
| Interne Links | `/reinigung-regensburg`, `/entruempelung-regensburg`, `/umzug-mit-reinigung`, `/leerfahrt-rueckfahrt`, `/anfrage-mit-preisrahmen`, `/buchung`. |
| CTA | Anfrage starten, Kosten einschaetzen, Fotos senden, Budget nennen. |
| Umsetzung | Bestehende Geldseite mit Autoritaetsmodulen staerken; Guides nur fuer komplexere Fragen. |
| Neue Seite? | Nein fuer einzelne Module. Ja nur fuer einen starken Guide wie Wohnungsuebergabe oder Umzugskosten, falls genug Substanz vorhanden ist. |
| Risiko | Startseite und Bayern-Seiten duerfen nicht gegen `/umzug-regensburg` um "Umzug Regensburg" konkurrieren. |
